# 週次レポートテンプレート

> Claude Code で週次レポートを作成する際に使用するテンプレート

---

## 使い方

Claude Code に以下のようにプロンプトを投げる:

```
今週の作業内容から週次レポートを作成してください。
テンプレートは eval_system/business_templates/weekly_report.md を参照。
- 今週やったこと: [箇条書き]
- 来週の予定: [箇条書き]
- 課題: [あれば]
```

---

## 標準週次レポートテンプレート

```markdown
# 週次レポート

| 項目 | 内容 |
|------|------|
| 報告者 | {名前} |
| 対象期間 | YYYY/MM/DD (月) - YYYY/MM/DD (金) |
| 報告日 | YYYY/MM/DD |

---

## 1. 今週の成果

### プロジェクト: {プロジェクト名1}
- {成果1}（進捗: XX%）
- {成果2}

### プロジェクト: {プロジェクト名2}
- {成果1}
- {成果2}

### その他
- {成果}

---

## 2. 数値サマリー

| 指標 | 今週 | 先週 | 増減 |
|------|------|------|------|
| 完了タスク数 | {N} | {N} | {+/-N} |
| PR作成数 | {N} | {N} | {+/-N} |
| レビュー数 | {N} | {N} | {+/-N} |
| 障害対応 | {N} | {N} | {+/-N} |

---

## 3. 課題・リスク

| # | 課題 | 影響度 | 対応状況 | 期限 |
|---|------|--------|---------|------|
| 1 | {課題内容} | 高/中/低 | {対応中/未対応} | MM/DD |
| 2 | {課題内容} | 高/中/低 | {対応中/未対応} | MM/DD |

---

## 4. 来週の計画

| # | タスク | プロジェクト | 優先度 | 見込み工数 |
|---|--------|------------|--------|----------|
| 1 | {タスク} | {PJ名} | 高/中/低 | {N}h |
| 2 | {タスク} | {PJ名} | 高/中/低 | {N}h |
| 3 | {タスク} | {PJ名} | 高/中/低 | {N}h |

---

## 5. 学習・改善

- 今週学んだこと: {内容}
- 改善したプロセス: {内容}
- 来週試したいこと: {内容}

---

## 6. 連絡事項

- {連絡事項}
```

## Claude Code 活用のコツ

### Git ログから自動生成

```
今週のgitコミットログから週次レポートの「今週の成果」セクションを作成して。
対象リポジトリ: <your-project>, <your-other-project>
期間: 今週月曜から今日まで
```

### GitHub Issues/PRs から生成

```
今週クローズしたIssueとマージしたPRから週次レポートを作成して。
gh issue list --state closed --since "2026-03-23"
gh pr list --state merged --since "2026-03-23"
```

### 前週レポートとの差分

```
先週のレポート（添付）と比較して、
来週の計画から今週の成果への達成状況をまとめてください。
```
