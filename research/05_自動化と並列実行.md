## 1. Headlessモード

> ソース: https://docs.anthropic.com/en/docs/claude-code/advanced

### 1.1 基本フラグ

| フラグ | 説明 |
|--------|------|
| `-p` / `--print` | 非対話（Headless）モードで実行。プロンプトを渡して結果を標準出力に返す |
| `--output-format` | 出力形式を指定: `text`（デフォルト）, `json`, `streaming-json` |
| `--session-id <id>` | セッションを指定して会話を継続 |
| `--resume` | 直前のセッションを再開 |
| `--continue` | 直前のセッションのコンテキストを引き継いで新規プロンプトを実行 |
| `--max-turns <n>` | エージェントループの最大ターン数を制限 |
| `--allowedTools` | 使用を許可するツールを指定 |
| `--disallowedTools` | 使用を禁止するツールを指定 |

### 1.2 使用例

```bash
# 基本的なHeadless実行
claude -p "このプロジェクトの構造を説明して"

# JSON形式で出力
claude -p "READMEを要約して" --output-format json

# ストリーミングJSON形式
claude -p "テスト追加して" --output-format streaming-json

# セッション継続
claude -p "前の続き" --session-id my-session-123

# ツール制限付き
claude -p "コードレビューして" --allowedTools "Read,Grep,Glob"

# ターン数制限
claude -p "バグを修正して" --max-turns 10
```

### 1.3 CI/CD環境での利用

CI/CD環境では以下の環境変数を設定して利用する:

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
# 非対話モードで自動承認
claude -p "npm testを実行して結果を報告" --allowedTools "Bash(npm:*),Read"
```

### 1.4 V2 Session API

> ソース: https://github.com/anthropics/claude-agent-sdk-demos （hello-world-v2）

V1の `query()` が全メッセージのAsync Generatorだったのに対し、V2は `send()` と `stream()` を分離し、マルチターン会話を実現する。

```typescript
// セッション作成
const session = unstable_v2_createSession({ model: 'sonnet' });

// メッセージ送信 + ストリーム取得
await session.send();
for await (const event of session.stream()) {
  // イベント処理
}

// セッション再開（永続化）
const resumed = unstable_v2_resumeSession(sessionId, { model: 'sonnet' });

// ワンショット（セッション不要の単発実行）
const result = unstable_v2_prompt('質問内容', { model: 'sonnet' });
```

**V1 vs V2 の違い:**

| 項目 | V1 (`query()`) | V2 (Session API) |
|------|----------------|-------------------|
| パターン | Async Generator | send() + stream() |
| 会話 | 単発プロンプト | マルチターン対応 |
| 永続化 | なし | sessionIdで再開可能 |
| ワンショット | query()で実行 | unstable_v2_prompt() |

## 2. GitHub Actions

> ソース: https://docs.anthropic.com/en/docs/claude-code/github-actions
> ソース: https://github.com/anthropics/claude-code-action

### 2.1 概要

`anthropics/claude-code-action` は、GitHub PR/Issueに対してClaude Codeを統合するGitHub Action。自動モード検出により、メンション応答・コードレビュー・実装支援を自動切替する。

- **Stars**: 6.7k / **License**: MIT / **Latest**: v1.0
- 認証: Anthropic API直接 / AWS Bedrock / Google Vertex AI / Microsoft Foundry

### 2.2 クイックスタート

```bash
claude /install-github-app
```

これにより以下が自動設定される:
- GitHub Appインストール
- シークレット設定
- リポジトリセットアップ

### 2.3 主要入力パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| `anthropic_api_key` | - | Anthropic APIキー |
| `claude_code_oauth_token` | - | OAuth認証トークン |
| `prompt` | - | カスタムプロンプト指示 |
| `trigger_phrase` | `@claude` | コメントトリガーフレーズ |
| `assignee_trigger` | - | ユーザー名トリガー |
| `label_trigger` | `claude` | ラベルトリガー |
| `track_progress` | - | 進捗トラッキングコメント有効化 |
| `claude_args` | - | Claude CLIへの追加引数 |
| `include_fix_links` | `true` | PRレビューの「Fix this」リンク |
| `use_sticky_comment` | `false` | 単一コメントに統合 |
| `classify_inline_comments` | `true` | インラインコメント分類 |
| `branch_prefix` | `claude/` | ブランチ名プレフィックス |
| `use_commit_signing` | - | コミット署名有効化 |
| `settings` | - | JSON設定文字列またはファイルパス |
| `additional_permissions` | - | 追加権限 |
| `allowed_bots` | - | 許可するbotリスト |
| `plugins` | - | プラグインリスト |
| `use_bedrock` | - | AWS Bedrock OIDC有効化 |
| `use_vertex` | - | Google Vertex AI OIDC有効化 |

### 2.4 出力パラメータ

### 2.6 ワークフロー例: 自動PRコードレビュー

```yaml
name: Claude Auto Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write
      id-token: write
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 1

      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: |
            REPO: ${{ github.repository }}
            PR NUMBER: ${{ github.event.pull_request.number }}

            Please review this pull request with a focus on:
            - Code quality and best practices
            - Potential bugs or issues
            - Security implications
            - Performance considerations

            Note: The PR branch is already checked out in the current working directory.

            Use `gh pr comment` for top-level feedback.
            Use `mcp__github_inline_comment__create_inline_comment` (with `confirmed: true`) to highlight specific code issues.
            Only post GitHub comments - don't submit review text as messages.

          claude_args: |
            --allowedTools "mcp__github_inline_comment__create_inline_comment,Bash(gh pr comment:*),Bash(gh pr diff:*),Bash(gh pr view:*)"
```

## 3. スケジュール実行

> ソース: https://github.com/shanraisshan/claude-code-best-practice

### 3.1 /loop（ローカル定期実行）

`/loop` コマンドでプロンプトやスラッシュコマンドをローカルで定期実行できる（最大3日間）。

```
/loop 5m /foo          # 5分間隔で /foo を実行
/loop 10m "check status"  # 10分間隔でステータス確認（デフォルト10分）
```

### 3.2 /schedule（クラウド定期実行 = Remote Trigger）

`/schedule` はAnthropicインフラ上でクラウド実行される。ローカルマシンがオフラインでも継続動作する。

```
/schedule              # スケジュール管理（作成・一覧・更新・削除）
```

cron式でスケジュール指定し、リモートエージェントとして実行される。

### 3.3 GitHub Actions cron

GitHub Actionsの `schedule` トリガーでcron実行可能:

```yaml
on:
  schedule:
    - cron: "0 0 * * 0"  # 毎週日曜0時
  workflow_dispatch:       # 手動実行も可能
```

### 3.4 Ralph Wiggum Loop Plugin

長時間タスク用の自律開発ループプラグイン。完了するまで繰り返し実行する。

## 4. Agent SDK

### 4.1 Python SDK

> ソース: https://github.com/anthropics/claude-agent-sdk-python

**インストール:**

```bash
pip install claude-agent-sdk
```

- Python 3.10+ 必須
- Claude Code CLIは自動バンドル（別途インストール不要）

**基本使用法 - query():**

```python
import anyio
from claude_agent_sdk import query

async def main():
    async for message in query(prompt="What is 2 + 2?"):
        print(message)

anyio.run(main)
```

**ClaudeSDKClient（双方向会話）:**

`ClaudeSDKClient` はカスタムツール・フックに対応した双方向インタラクティブセッションを提供。`query()` では利用できない機能を含む。

**カスタムツール（インプロセスMCPサーバー）:**

```python
from claude_agent_sdk import tool, create_sdk_mcp_server

@tool("greet", "Greet a user", {"name": str})
async def greet_user(args):
    return {
        "content": [{"type": "text", "text": f"Hello, {args['name']}!"}]
    }

server = create_sdk_mcp_server(
    name="my-tools",
    version="1.0.0",
    tools=[greet_user]
)
```

利点:
- サブプロセス管理が不要
- IPC オーバーヘッド削減
- 型安全な関数呼び出し

**フック（エージェント制御）:**

```python
async def check_bash_command(input_data, tool_use_id, context):
    if input_data["tool_name"] != "Bash":
        return {}
    # コマンドの安全性を評価...
    return {"hookSpecificOutput": {...}}
```

`PreToolUse` フックでツール使用前に介入し、許可・ブロックを制御可能。

**ツール権限管理:**

- `allowed_tools`: リストされたツールは自動承認
- `disallowed_tools`: 明示的にブロック
- 未リストのツールは `permission_mode` と `can_use_tool` で判定

**エラーハンドリング:**

| 例外クラス | 説明 |
|-----------|------|
| `ClaudeSDKError` | 基底クラス |
| `CLINotFoundError` | Claude Code未インストール |
| `CLIConnectionError` | 接続失敗 |
| `ProcessError` | プロセス終了（exit code付き） |
| `CLIJSONDecodeError` | レスポースパース失敗 |

**混合サーバーサポート:**

SDK MCPサーバー（インプロセス）と外部MCPサーバーを同時に登録・併用可能。

```typescript
import { query } from '@anthropic-ai/claude-agent-sdk';

const q = query({
  prompt: 'Your prompt here',
  options: {
    maxTurns: 100,        // 最大ターン数
    cwd: './workspace',   // 作業ディレクトリ
    model: 'sonnet',      // モデル指定
    executable: 'node',   // 実行バイナリ
    allowedTools: ['Read', 'Write', 'Edit', 'Bash', 'Glob', 'Grep']

for await (const message of q) {
  if (message.type === 'assistant' && message.message) {
    const textContent = message.message.content.find(
      (c: any) => c.type === 'text'
    );
    if (textContent && 'text' in textContent) {

    }
  }
}
```

## 5. 並列実行

> ソース: https://github.com/shanraisshan/claude-code-best-practice
> ソース: https://github.com/affaan-m/everything-claude-code

### 5.1 Git Worktree

各エージェントに独立した作業コピーを与え、同一リポジトリで複数Claude Codeセッションを同時実行する。

```bash
# worktreeを作成
git worktree add ../feature-auth feature/auth
git worktree add ../feature-api feature/api

# 各worktreeで別々のClaude Codeを起動
cd ../feature-auth && claude
cd ../feature-api && claude
```

メリット:
- ブランチ間の干渉なし
- 各エージェントが独立したファイルシステムで動作
- 同一リポジトリのgitヒストリーを共有

### 5.2 Cascade Method

段階的にタスクを分解し、複数のClaude Codeインスタンスにカスケード（連鎖）させる手法。

1. **計画フェーズ**: 1つ目のインスタンスでタスク分解・計画策定
2. **実行フェーズ**: 計画に基づき複数インスタンスを並列起動
3. **統合フェーズ**: 結果をマージ

### 5.3 /fork コマンド

現在のコンテキストをフォークして新しいエージェントコンテキストを生成する。Skills の「progressive disclosure」と連携し、コンテキストを分岐させて並列作業を実現。

### 5.4 2インスタンスキックオフパターン

ターミナル（iTerm/Ghostty/tmux推奨、IDE非推奨）で複数Claude Codeセッションを同時起動:

```bash
# ターミナル1
claude   # /rename backend

# ターミナル2
claude   # /rename frontend
```

- `/rename` でインスタンスにラベル付け
- `/resume` で中断セッションの再開
- IDEではなくターミナルベースを推奨（パフォーマンス上の理由）

### 5.5 Agent Teams

複数エージェントが同一コードベースで並列作業し、共有タスク調整を行う。ビルトイン環境変数により隔離コンテキスト間での並行タスク実行が可能。

### 5.6 サブエージェント委譲

```
"use subagents" と指示してより多くのコンピュートを問題に投入
```

タスクをサブエージェントにオフロードし、メインコンテキストを集中させる。テスト時コンピュートを別コンテキストウィンドウで強化。

## 7. マルチモデル並列・トークン予算管理

> ソース: https://github.com/shanraisshan/claude-code-best-practice
> ソース: https://github.com/affaan-m/everything-claude-code

### 7.1 クロスモデルワークフロー

Claude CodeとCodex CLIを組み合わせ、計画/実装レビューを同時に行うマルチモデル並列分析。

### 7.2 モデル選択戦略

```json
{
  "model": "sonnet",
  "env": {
    "MAX_THINKING_TOKENS": "10000",
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "50",
    "CLAUDE_CODE_SUBAGENT_MODEL": "haiku"
  }
}
```

| 用途 | モデル | コスト削減 |
|------|--------|-----------|
| ルーティン作業 | `/model sonnet` | 約60%削減 |
| 複雑なアーキテクチャ/デバッグ/深い推論 | `/model opus` | - |
| サブエージェント | `haiku` | 最大コスト効率 |

### 7.3 トークン予算管理

**コンテキスト監視:**
- `/context` でトークン使用量を確認
- `/compact` は最大50%時点で実行（「agent dumb zone」回避）
- compactionエラー時は1Mトークンモデルに切替後に再compact

**セッション管理:**
- `/clear` でタスク切替時にコンテキストリセット
- `/extra-usage` でオーバーフロー課金設定（拡張トークン予算）
- チェックポイント機能（`Esc Esc` or `/rewind`）でgitベースの編集追跡

**MCP制約:**
- プロジェクトあたり **10 MCP以下** を推奨
- アクティブツール **80以下** を推奨
- 超過時: コンテキストウィンドウが200k → 約70kに縮小

### 7.4 AgentShield（セキュリティスキャン in CI/CD）

```bash
npx ecc-agentshield scan --fix           # 安全な問題を自動修正
npx ecc-agentshield scan --opus --stream  # 深い分析
```

CI/CDビルドゲートとして、クリティカル検出時にexit code 2を返す。
