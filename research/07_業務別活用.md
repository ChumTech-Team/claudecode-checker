## 1. システム開発ワークフロー

### 基本フロー: Research → Plan → Execute → Review → Ship

Claude Codeを活用した開発ワークフローは、以下の5段階に収束する（shanraisshan/claude-code-best-practice より）。

#### Step 1: コードベース理解（Research）
- 新規プロジェクト参加時にClaude Codeへ「このコードベースの全体構造を説明して」と指示
- `@path` インポートやCLAUDE.mdファイルによるコンテキスト永続化
- Anthropic社内では、Data Infrastructureチームが複雑なKubernetesクラスタ障害時にスクリーンショットをClaude Codeに投入し、Google CloudのUIを通じてPod IPアドレス枯渇問題を解決（ネットワーク専門家不要）

#### Step 2: 実装（Execute）
- Plan Modeで必ず開始し、フェーズごとにゲート付き計画を作成
- 各フェーズにユニットテスト、自動テスト、統合テストを含める
- TDD-first（Superpowers: 118kスター）、Spec-driven（Spec Kit: 83kスター）、Wave実行（Get Shit Done: 43kスター）など複数パターン
- サブエージェント: 独立したコンテキスト内で動作する自律アクター。`.claude/agents/<name>.md`に定義

#### Step 3: テスト
- Claudeに「この変更について厳しく質問して」「これが動くことを証明して」と指示
- マルチエージェントPR分析でバグとセキュリティ問題を検出
- クロスモデルワークフロー: Claude CodeとCodexを組み合わせた専門コードレビュー

#### Step 4: レビュー
- Code Review機能: マルチエージェントPR分析
- セカンダリClaude instanceを「スタッフエンジニアレベルの計画レビュアー」として起動
- 「最初のfix が凡庸だったら、エレガントな解決策を要求する」

#### Step 5: デプロイ（Ship）
- `/loop` でローカル定期実行（最大3日）
- `/schedule` でクラウド実行（cronスケジュール）
- CI/CDパイプラインとの統合

### 主要コミュニティフレームワーク比較

| フレームワーク | GitHubスター | 特徴 |
|---|---|---|
| Superpowers | 118k | TDD-first、全体計画レビュー |
| Everything Claude Code | 111k | インスティンクトスコアリング、多言語ルール |
| Spec Kit | 83k | Spec駆動、22+統合ツール |
| gstack | 52k | ロールペルソナ、並列スプリント実行 |
| Get Shit Done | 43k | Wave実行、200Kトークンの新鮮コンテキスト |

## 2. リサーチと調査

### Search-Firstプロトコル
- WebSearch / WebFetchツールを活用した情報収集
- 市場調査: 競合分析、価格調査、業界トレンド
- 技術調査: API調査、アーキテクチャ比較、PoC

### Anthropic社内での活用
- Growth Marketingチーム: CSVファイルから数百の既存広告を処理し、パフォーマンス不良広告を特定、2つの専門サブエージェントで数分で数百の新広告バリエーションを生成
- エンジニアの80%以上がClaude Codeを日常的に使用

### 推奨構成
- MCPサーバーでWeb検索、データベース接続を統合
- `.mcp.json` でプロトコルサーバーを設定
- サブエージェントを調査専門として分離

## 3. 資料作成

### 対応フォーマット（Agent Skills）
Anthropicの公式Skillsリポジトリは以下のドキュメントスキルを提供:

| スキル | 説明 |
|---|---|
| **DOCX** | Word文書の作成・編集 |
| **PDF** | PDFの解析・フォームフィールド抽出 |
| **PPTX** | PowerPointプレゼンテーション作成 |
| **XLSX** | Excelスプレッドシート作成・編集 |

> **注意**: ドキュメントスキルは「source-available（ソース公開）」だがオープンソースではない。Claudeの本番ドキュメント機能を動かすリファレンス実装として共有。

### スキルの構造
```
skills/
├── Creative & Design      # アート、音楽、デザイン
├── Development & Technical # Webアプリテスト、MCPサーバー生成
├── Enterprise & Communication # コミュニケーション、ブランディング
└── Document Skills        # DOCX, PDF, PPTX, XLSX
```

### スキルの定義方法
`SKILL.md` ファイルに以下のフロントマターを記述:
```yaml
---
name: my-skill-name
description: スキルの説明と使用タイミング
---
```

### Claude Codeでの登録・利用
```bash
/plugin marketplace add anthropics/skills
/plugin install document-skills@anthropic-agent-skills
```
使用例: 「PDF skillを使って `path/to/file.pdf` のフォームフィールドを抽出して」

**ソース**: https://github.com/anthropics/skills

## 4. クライアントコミュニケーション

### メール作成
- Enterprise & Communication スキルカテゴリで対応
- internal-comms（社内コミュニケーション）スキル: 組織内メール、通知文の作成
- doc-coauthoring（文書共著）スキル: 提案書・報告書の共同執筆

### 議事録作成
- 会議音声の文字起こし結果をClaude Codeに投入
- 要点抽出、アクションアイテム整理、フォローアップメール生成
- Commandsとして `.claude/commands/` に定型ワークフローを保存

### Slack連携
- Channels機能（Beta）: Telegram/DiscordからのプッシュイベントをClaude Codeセッションに注入
- MCPサーバー経由でSlack APIと連携可能

### 非技術者の活用事例（Anthropic社内）
- コーディング経験ゼロの社員が「このダッシュボードをクエリして、情報を取得し、クエリを実行して、Excel出力を作成」と指示するだけで全ワークフローを実行
- Legalチーム: アクセシビリティツールや部門間調整システムを従来のエンジニアリングサポートなしで構築

**ソース**: https://www.anthropic.com/news/how-anthropic-teams-use-claude-code

## 5. チームマネジメント

### 進捗レポート
- Claude Codeにプロジェクトの状態を分析させ、自動レポート生成
- Git履歴からPR数、コミット頻度、セッション期間を集計
- Linear連携による自動レポート配信

### タスク管理
- Agent Teams機能: 複数エージェントが共有タスク調整で並列作業
- `.claude/commands/` にタスク管理ワークフローを定義
- ステータスラインでコンテキスト使用量、モデル選択、セッションコストを表示

### 1on1
- 過去のコミット・PR履歴を分析してパフォーマンスサマリーを生成
- チームメンバーの採用トレンド、生産性比較を可視化

## 6. ドキュメント管理

### Notion連携
- Notion MCP サーバーまたは ntn CLI（公式CLI）を使用
- ページ・DB・ブロックの操作、検索、ファイルアップロード

### Google Drive連携
- Google Drive MCPサーバー経由でファイル操作
- 一覧・検索・アップロード・ダウンロード・作成・移動・削除

### メモリシステム
- CLAUDE.mdファイルによるコンテキスト永続化
- 階層的ルール構成:
  - `~/.claude/rules/` - グローバルルール
  - `.claude/rules/` - プロジェクト固有ルール
  - `CLAUDE.md` - プロジェクトメモリ

## 7. 各業務の推奨MCP/Skills/Plugins構成

### プロジェクト構成例
```
.claude/
├── agents/          # サブエージェント定義
│   ├── researcher.md
│   ├── reviewer.md
│   └── writer.md
├── commands/        # ワークフローオーケストレーター
│   ├── daily-report.md
│   ├── code-review.md
│   └── meeting-notes.md
├── skills/          # 再利用可能な知識モジュール
│   └── <name>/SKILL.md
├── hooks/           # イベントハンドラ（スクリプト、HTTP、プロンプト）
├── settings.json    # 設定（権限モード、モデル、出力スタイル）
└── rules/           # 階層的ルール
CLAUDE.md            # プロジェクトメモリ
.mcp.json            # MCPサーバー設定
```

### 業務別推奨構成

| 業務 | 推奨MCP/ツール | スキル | コマンド |
|---|---|---|---|
| システム開発 | GitHub, Linear | document-skills | code-review, test-run |
| リサーチ | WebSearch, WebFetch | - | research-report |
| 資料作成 | Google Drive, Notion | PPTX, DOCX, XLSX, PDF | create-proposal |
| コミュニケーション | Slack, Gmail | internal-comms | meeting-notes, email-draft |
| チーム管理 | Linear, GitHub | - | daily-report, sprint-review |
| ドキュメント管理 | Notion, Google Drive | doc-coauthoring | sync-docs |

## 8. 非コーディング用途（Cowork）

### Anthropic社内の非技術チーム活用

Claude Codeは「コーディングツール」を超え、あらゆる業務に適用可能:

1. **Growth Marketing**: 広告分析・生成ワークフロー（CSVインポート → パフォーマンス分析 → 新バリエーション生成）
2. **Legal**: アクセシビリティツール構築、部門間調整システム
3. **Data Infrastructure**: インフラ障害対応（Kubernetesトラブルシューティング）
4. **非技術者全般**: ダッシュボードクエリ → データ取得 → レポート生成の一気通貫

### Hot Features（Beta/最新）

| 機能 | 説明 |
|---|---|
| **Auto Mode** | バックグラウンド安全分類器で手動許可プロンプトを置換 |
| **Channels** | Telegram/Discordからセッションにイベントをプッシュ |
| **Code Review** | マルチエージェントPR分析 |
| **Scheduled Tasks** | `/loop`（ローカル、最大3日）、`/schedule`（クラウド） |
| **Voice Dictation** | Push-to-talk入力、20言語対応 |
| **Agent Teams** | 複数エージェントの並列作業・タスク調整 |
| **Remote Control** | `/rc` コマンドで任意デバイスからセッション継続 |

### プロンプティングのベストプラクティス
- マイクロマネジメントせず、チャレンジする: 「変更点について厳しく質問して」「これが動くことを証明して」
- 凡庸なfixの後にエレガントな解決策を要求
- Plan Modeで必ず開始
