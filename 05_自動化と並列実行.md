## 1. Headlessモード

### R-001: `-p` / `--print` フラグで非対話モードを起動する

```bash
claude -p "このプロジェクトの構造を説明して"
```

### R-002: `--output-format` で出力形式を指定する

`text`（デフォルト）、`json`、`streaming-json` の3種がある。

```bash
claude -p "READMEを要約して" --output-format json
claude -p "テスト追加して" --output-format streaming-json
```

### R-003: `--session-id` でセッションを指定して会話を継続する

```bash
claude -p "前の続き" --session-id my-session-123
```

### R-004: `--resume` は直前のセッションを再開、`--continue` は直前のコンテキストを引き継いで新規プロンプトを実行する

### R-005: `--max-turns` でエージェントループの最大ターン数を制限する

```bash
claude -p "バグを修正して" --max-turns 10
```

### R-006: `--allowedTools` / `--disallowedTools` で使用ツールを制御する

```bash
claude -p "コードレビューして" --allowedTools "Read,Grep,Glob"
```

### R-007: CI/CD環境では `ANTHROPIC_API_KEY` を環境変数に設定し、`--allowedTools` でツールを制限する

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
claude -p "npm testを実行して結果を報告" --allowedTools "Bash(npm:*),Read"
```

### R-008: V2 Session API は `send()` + `stream()` を分離し、マルチターン会話を実現する

V1の `query()` が全メッセージのAsync Generatorだったのに対し、V2はセッション概念を持つ。

```typescript
const session = unstable_v2_createSession({ model: 'sonnet' });
await session.send();
for await (const event of session.stream()) { /* イベント処理 */ }
```

### R-009: V2 Session API は `sessionId` でセッションを永続化・再開できる

```typescript
const resumed = unstable_v2_resumeSession(sessionId, { model: 'sonnet' });
```

### R-010: V2のワンショット実行には `unstable_v2_prompt()` を使う

```typescript
const result = unstable_v2_prompt('質問内容', { model: 'sonnet' });
```

### R-011: V1 vs V2 の主な違い

| 項目 | V1 (`query()`) | V2 (Session API) |
|------|----------------|-------------------|
| パターン | Async Generator | send() + stream() |
| 会話 | 単発プロンプト | マルチターン対応 |
| 永続化 | なし | sessionIdで再開可能 |
| ワンショット | query()で実行 | unstable_v2_prompt() |

**情報源:** https://docs.anthropic.com/en/docs/claude-code/advanced, https://github.com/anthropics/claude-agent-sdk-demos

### R-017: 自動PRコードレビューは `pull_request` の `opened` / `synchronize` イベントで起動する

必要な permissions: `contents: read`, `pull-requests: write`, `id-token: write`

```yaml
name: Claude Auto Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write
      id-token: write
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 1
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: |
            REPO: ${{ github.repository }}
            PR NUMBER: ${{ github.event.pull_request.number }}
            Please review this pull request...
          claude_args: |
            --allowedTools "mcp__github_inline_comment__create_inline_comment,Bash(gh pr comment:*),Bash(gh pr diff:*),Bash(gh pr view:*)"
```

### R-042: `@tool` デコレータと `create_sdk_mcp_server` でインプロセスMCPサーバーを作成する

```python
from claude_agent_sdk import tool, create_sdk_mcp_server

@tool("greet", "Greet a user", {"name": str})
async def greet_user(args):
    return {"content": [{"type": "text", "text": f"Hello, {args['name']}!"}]}

server = create_sdk_mcp_server(name="my-tools", version="1.0.0", tools=[greet_user])
```

利点: サブプロセス管理不要、IPCオーバーヘッド削減、型安全な関数呼び出し。

### R-048: TypeScript SDK の `query()` でストリーミング実行する

```typescript
import { query } from '@anthropic-ai/claude-agent-sdk';

const q = query({
  prompt: 'Your prompt here',
  options: {
    maxTurns: 100,
    cwd: './workspace',
    model: 'sonnet',
    executable: 'node',
    allowedTools: ['Read', 'Write', 'Edit', 'Bash', 'Glob', 'Grep']
  }
});

for await (const message of q) {
  if (message.type === 'assistant' && message.message) {
    const textContent = message.message.content.find((c: any) => c.type === 'text');
    if (textContent && 'text' in textContent) {
      console.log(textContent.text);
    }
  }
}
```

## 5. 並列実行

### R-055: Git Worktreeで各エージェントに独立した作業コピーを与える

```bash
git worktree add ../feature-auth feature/auth
git worktree add ../feature-api feature/api

# 各worktreeで別々のClaude Codeを起動
cd ../feature-auth && claude
cd ../feature-api && claude
```

メリット: ブランチ間の干渉なし、独立したファイルシステム、同一リポジトリのgitヒストリー共有。

### R-056: Cascade Methodは計画・実行・統合の3フェーズでタスクを分解し並列実行する

1. **計画フェーズ**: 1つ目のインスタンスでタスク分解・計画策定
2. **実行フェーズ**: 計画に基づき複数インスタンスを並列起動
3. **統合フェーズ**: 結果をマージ

### R-057: `/fork` で現在のコンテキストをフォークして新しいエージェントコンテキストを生成する

Skills の「progressive disclosure」と連携し、コンテキストを分岐させて並列作業を実現。

### R-058: 2インスタンスキックオフパターンではターミナルで複数セッションを同時起動する

```bash
# ターミナル1
claude   # /rename backend

# ターミナル2
claude   # /rename frontend
```

- `/rename` でインスタンスにラベル付け
- `/resume` で中断セッションの再開
- IDEではなくターミナルベースを推奨（パフォーマンス上の理由）

### R-059: Agent Teams で複数エージェントが同一コードベースで並列作業する

ビルトイン環境変数により隔離コンテキスト間での並行タスク実行が可能。

### R-060: 「use subagents」と指示してサブエージェントにタスクを委譲する

タスクをサブエージェントにオフロードし、メインコンテキストを集中させる。テスト時コンピュートを別コンテキストウィンドウで強化。

## automation-evaluator用 評価チェックリスト

以下のチェックリストは、Claude Codeの自動化・並列実行設定を評価する際に使用する。

### A. Headlessモード (R-001 -- R-011)

- [ ] `-p` フラグで非対話モードを正しく使用しているか
- [ ] `--output-format` が用途に適切か（CI/CDでは `json` または `streaming-json` 推奨）
- [ ] `--max-turns` でエージェントループに上限を設けているか
- [ ] `--allowedTools` / `--disallowedTools` で最小権限を適用しているか
- [ ] CI/CD環境で `ANTHROPIC_API_KEY` が安全に管理されているか
- [ ] V2 Session APIを使用する場合、マルチターン・永続化の要否を検討したか

### B. GitHub Actions (R-012 -- R-034)

- [ ] `anthropics/claude-code-action@v1` を使用しているか
- [ ] `permissions` が必要最小限に設定されているか
- [ ] `trigger_phrase` / `label_trigger` が適切に設定されているか
- [ ] PRレビューワークフローで `fetch-depth: 1` を指定しているか（週次メンテナンスは `fetch-depth: 0`）
- [ ] `--allowedTools` でツールを用途に応じて制限しているか
- [ ] セキュリティレビューでOWASP Top 10を網羅しているか
- [ ] 構造化出力（`--json-schema`）を活用して後続ステップと連携しているか
- [ ] MCPサーバー設定でシークレットを `env` 経由で注入しているか（ハードコード禁止）
- [ ] モード検出（Interactive / Automation）を理解して設計しているか

### C. スケジュール実行 (R-035 -- R-038)

- [ ] `/loop`（ローカル、最大3日）と `/schedule`（クラウド、オフライン対応）を用途に応じて使い分けているか
- [ ] GitHub Actions cronに `workflow_dispatch` を併用して手動実行も可能にしているか

### D. Agent SDK (R-039 -- R-054)

- [ ] Python SDK / TypeScript SDK のインストール要件を満たしているか
- [ ] `query()` と `ClaudeSDKClient` の使い分けが適切か
- [ ] カスタムツールをインプロセスMCPサーバーとして実装しているか
- [ ] `PreToolUse` フックでツール使用前の制御を実装しているか
- [ ] ツール権限（`allowed_tools` / `disallowed_tools`）を適切に設定しているか
- [ ] エラーハンドリングで適切な例外クラスをキャッチしているか
- [ ] デモプロジェクトが本番環境に流用されていないか

### E. 並列実行 (R-055 -- R-060)

- [ ] Git Worktreeで各エージェントに独立した作業コピーを与えているか
- [ ] Cascade Method（計画・実行・統合）の3フェーズを踏んでいるか
- [ ] ターミナルベースで実行しているか（IDE非推奨）
- [ ] `/rename` でインスタンスにラベルを付けて識別しているか
- [ ] サブエージェント委譲でメインコンテキストの負荷を軽減しているか

### F. オーケストレーター (R-061 -- R-068)

- [ ] プロジェクト規模に適したオーケストレーターを選択しているか
- [ ] Command / Agent / Skill の3層階層を理解して使い分けているか
- [ ] `ccg-workflow` ランタイムの初期化を行っているか（`/orchestrate` 使用時）

### G. マルチモデル・トークン管理 (R-069 -- R-077)

- [ ] 用途に応じたモデル選択（sonnet / opus / haiku）をしているか
- [ ] `/compact` を50%以前に実行してコンテキスト劣化を防いでいるか
- [ ] タスク切替時に `/clear` でコンテキストをリセットしているか
- [ ] MCPサーバー数が10以下、アクティブツール数が80以下を遵守しているか
- [ ] AgentShieldをCI/CDビルドゲートに組み込んでいるか
