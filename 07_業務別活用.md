# 07. 業務別活用ナレッジベース

> business-use-evaluator 用ナレッジベース
> 対象: 社長業（開発・リサーチ・資料作成・クライアント対応・マネジメント）

---

## 1. システム開発ワークフロー

### 1-1. 基本5段階フロー

| フェーズ | 概要 |
|---|---|
| Research | コードベース理解。`@path` インポートやCLAUDE.mdでコンテキスト永続化 |
| Execute | Plan Modeで開始し、フェーズごとにゲート付き計画を作成 |
| Test | 「この変更について厳しく質問して」「動くことを証明して」と指示 |
| Review | マルチエージェントPR分析。セカンダリinstanceをレビュアーとして起動 |
| Ship | `/loop`（ローカル最大3日）、`/schedule`（クラウドcron）、CI/CD統合 |

**R-001**: 新規プロジェクト参加時は「このコードベースの全体構造を説明して」と最初に指示する。

**R-002**: Plan Modeで必ず開始し、各フェーズにユニットテスト・自動テスト・統合テストを含める。

**R-003**: サブエージェントは `.claude/agents/<name>.md` に定義する。独立コンテキストで動作する自律アクター。

**R-004**: レビューでは「最初のfixが凡庸だったら、エレガントな解決策を要求する」。

**R-005**: クロスモデルワークフロー（Claude CodeとCodexの組み合わせ等）で専門コードレビューを行える。

**R-006**: Anthropic社内ではスクリーンショットをClaude Codeに投入し、Kubernetes Pod IPアドレス枯渇問題をネットワーク専門家不要で解決した実績がある。

### 1-2. コミュニティフレームワーク

| フレームワーク | GitHubスター | 特徴 |
|---|---|---|
| Superpowers | 118k | TDD-first、全体計画レビュー |
| Everything Claude Code | 111k | インスティンクトスコアリング、多言語ルール |
| Spec Kit | 83k | Spec駆動、22+統合ツール |
| gstack | 52k | ロールペルソナ、並列スプリント実行 |
| Get Shit Done | 43k | Wave実行、200Kトークンの新鮮コンテキスト |

**R-007**: TDD-firstならSuperpowers、Spec駆動ならSpec Kit、大量タスクの並列処理ならgstackを選択する。

### 1-3. 推奨MCP/Skills/Plugins構成

| 項目 | 推奨 |
|---|---|
| MCP | GitHub, Linear |
| スキル | document-skills |
| コマンド | code-review, test-run |

**R-008**: システム開発ではGitHub MCPとLinear MCPを `.mcp.json` に設定する。

**R-009**: コードレビューとテスト実行は `.claude/commands/` にワークフローとして定義する。

**情報源**: https://github.com/shanraisshan/claude-code-best-practice

## 2. リサーチと調査

### 2-1. Search-Firstプロトコル

**R-010**: リサーチはWebSearch / WebFetchツールを活用した情報収集から始める。

**R-011**: 市場調査（競合分析・価格調査・業界トレンド）と技術調査（API調査・アーキテクチャ比較・PoC）を用途に応じて使い分ける。

**R-012**: MCPサーバーでWeb検索、データベース接続を統合し、`.mcp.json` でプロトコルサーバーを設定する。

**R-013**: サブエージェントを調査専門として分離する（`.claude/agents/researcher.md`）。

### 2-2. Anthropic社内活用事例

**R-014**: Growth Marketingチームは、CSVから数百の広告を処理し、パフォーマンス不良広告を特定、2つの専門サブエージェントで数分で数百の新バリエーションを生成した。

**R-015**: Anthropicではエンジニアの80%以上がClaude Codeを日常的に使用している。

### 2-3. 推奨MCP/Skills/Plugins構成

| 項目 | 推奨 |
|---|---|
| MCP | WebSearch, WebFetch |
| スキル | - |
| コマンド | research-report |

**R-016**: リサーチ結果レポートは `.claude/commands/research-report.md` に定型化する。

**情報源**: https://www.anthropic.com/news/how-anthropic-teams-use-claude-code

## 3. 資料作成

### 3-1. 対応フォーマット（Agent Skills）

| スキル | 説明 |
|---|---|
| DOCX | Word文書の作成・編集 |
| PDF | PDFの解析・フォームフィールド抽出 |
| PPTX | PowerPointプレゼンテーション作成 |
| XLSX | Excelスプレッドシート作成・編集 |

**R-017**: ドキュメントスキルは「source-available（ソース公開）」だがオープンソースではない。Claudeの本番ドキュメント機能のリファレンス実装。

**R-018**: スキルは `SKILL.md` ファイルにフロントマターで定義する。

```yaml
---
name: my-skill-name
description: スキルの説明と使用タイミング
---
```

### 3-2. スキルのディレクトリ構造

```
skills/
├── Creative & Design        # アート、音楽、デザイン
├── Development & Technical  # Webアプリテスト、MCPサーバー生成
├── Enterprise & Communication # コミュニケーション、ブランディング
└── Document Skills          # DOCX, PDF, PPTX, XLSX
```

### 3-3. Claude Codeでの登録・利用

```bash
/plugin marketplace add anthropics/skills
/plugin install document-skills@anthropic-agent-skills
```

**R-019**: 資料作成時は「PDF skillを使って `path/to/file.pdf` のフォームフィールドを抽出して」のようにスキル名を明示して指示する。

### 3-4. 推奨MCP/Skills/Plugins構成

| 項目 | 推奨 |
|---|---|
| MCP | Google Drive, Notion |
| スキル | PPTX, DOCX, XLSX, PDF |
| コマンド | create-proposal |

**R-020**: 提案書作成は `.claude/commands/create-proposal.md` にワークフローを定義する。

**R-021**: Google DriveやNotion MCPと組み合わせて資料の保存・共有まで一気通貫で行う。

**情報源**: https://github.com/anthropics/skills

## 7. プロジェクト構成テンプレート

**R-045**: 以下のディレクトリ構成をプロジェクトの標準とする。

```
.claude/
├── agents/          # サブエージェント定義
│   ├── researcher.md
│   ├── reviewer.md
│   └── writer.md
├── commands/        # ワークフローオーケストレーター
│   ├── daily-report.md
│   ├── code-review.md
│   └── meeting-notes.md
├── skills/          # 再利用可能な知識モジュール
│   └── <name>/SKILL.md
├── hooks/           # イベントハンドラ（スクリプト、HTTP、プロンプト）
├── settings.json    # 設定（権限モード、モデル、出力スタイル）
└── rules/           # 階層的ルール
CLAUDE.md            # プロジェクトメモリ
.mcp.json            # MCPサーバー設定
```

**R-046**: サブエージェントは researcher / reviewer / writer の3種を基本とする。

**R-047**: `.mcp.json` にプロジェクトで使用する全MCPサーバーを集約定義する。

## 8. 業務横断: 非コーディング用途（Cowork）

### 8-1. 非技術チーム活用パターン

| チーム | 活用内容 |
|---|---|
| Growth Marketing | CSV広告分析 → パフォーマンス分析 → 新バリエーション生成 |
| Legal | アクセシビリティツール構築、部門間調整システム |
| Data Infrastructure | Kubernetesトラブルシューティング |
| 非技術者全般 | ダッシュボードクエリ → データ取得 → レポート生成 |

**R-048**: Claude Codeは「コーディングツール」ではなくあらゆる業務に適用可能と認識する。

### 8-2. Hot Features（Beta/最新）

| 機能 | 説明 |
|---|---|
| Auto Mode | バックグラウンド安全分類器で手動許可プロンプトを置換 |
| Channels | Telegram/Discordからセッションにイベントをプッシュ |
| Code Review | マルチエージェントPR分析 |
| Scheduled Tasks | `/loop`（ローカル最大3日）、`/schedule`（クラウド） |
| Voice Dictation | Push-to-talk入力、20言語対応 |
| Agent Teams | 複数エージェントの並列作業・タスク調整 |
| Remote Control | `/rc` コマンドで任意デバイスからセッション継続 |

**R-049**: `/loop` はローカルで最大3日の定期実行、`/schedule` はクラウドcronスケジュール。

**R-050**: Voice Dictationは20言語対応のPush-to-talk入力。

**R-051**: Agent Teamsで複数エージェントが並列作業・タスク調整を行う。

**R-052**: Remote Controlの `/rc` で任意デバイスからセッション継続が可能。

### 8-3. プロンプティングのベストプラクティス

**R-053**: マイクロマネジメントせず、チャレンジする。「変更点について厳しく質問して」「これが動くことを証明して」。

**R-054**: 凡庸なfixの後にエレガントな解決策を要求する。

**R-055**: Plan Modeで必ず開始する。
